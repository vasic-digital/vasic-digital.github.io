Крючок

За каждым надёжным инструментом автоматизации стоит основа, которую никто не видит. Server Factory Core Framework является именно такой основой — многократно используемым Kotlin-ядром, которое питает инструменты, такие как Mail Server Factory, и даёт всему экосистеме Server Factory единую, последовательную основу для построения серверных приложений.

Почему это fascинirует

Большинство инструментов автоматизации инфраструктуры представляют собой кучу скриптов, которые со временем разбегаются. Server Factory подошёл к этому вопросу с другой стороны: извлёк общую основу — модульный дизайн, централизованное логирование, общие утилиты, управление вверх по течению — в Core Framework, а затем построил каждый инструмент подготовки на его основе. Именно поэтому инструменты экосистемы кажутся целостными: они наследуют одни и те же архитектурные шаблоны и надёжную структуру. Это применение подхода фреймворка к не очень привлекательному, но критически важному миру развертывания серверов.

Трудные проблемы

Разработка фреймворка, на котором могут строиться несколько различных инструментов, фундаментально сложнее, чем создание любого отдельного инструмента. Вам необходимо предвидеть, что серверные приложения будут иметь общего — логирование, которое ведёт себя последовательно, утилиты, которые не допускают ошибок, модель управления вверх по течению, которая подходит многим сервисам — без чрезмерного ограничения инструментов, которые от вас зависят. Если абстракции неправильны, каждому потребительскому инструменту наследуется ошибка; если они правильны, вы ускоряете все инструменты одновременно. Поддержание этой основы при росте экосистемы, сохраняя потребителей стабильными, — это постоянная дисциплина.

Что делает его революционным

Core Framework является рычагом. Поскольку сложные, общие части находятся в одном месте, каждый новый завод — для почтовых сервисов, веб-сервисов, кэширующих прокси, SonarQube и многое другое — начинает с проверенной основы, а не с пустого файла. Последовательность во всей экосистеме означает меньшее обслуживание, меньше сюрпризов и единый опыт эксплуатации, независимо от того, что вы развертываете. Как открытый исходный код, он также документирует чистый, многократно используемый шаблон для всех, кто строит свои собственные инструменты автоматизации серверов: разделяет общий движок и конкретный инструмент.

Как мы решили самые трудные части

Фреймворк намеренно концентрирует поперечные проблемы — логирование, утилиты, модульную структуру и управление вверх по течению — так, чтобы отдельные заводы никогда не реализовывали их заново. Этот модульный дизайн является механизмом, который сохраняет потребителей последовательными: они составляют компоненты фреймворка, а не копируют их. Доказательством является сама экосистема. Mail Server Factory и его братские заводы стоят на этой основе, что означает, что абстракции фреймворка были проверены реальными, производственными инструментами подготовки, а не придуманы в изоляции. Ценность фреймворка измеряется не функциями, а тем, сколько он позволяет инструментам выше него пропустить.

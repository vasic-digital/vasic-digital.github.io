## Қармақ

Әрбір сенімді автоматтандыру құралының артында көзге көрінбейтін негіз жатыр. Server Factory Core Framework осы негіз – бүкіл Server Factory экосистемасына біртұтас, тұрақты арқау болып, серверлік қолданбаларды құруға арналған қайта қолданылатын Kotlin қозғалтқышы. Оның үстінде Mail Server Factory сияқты құралдар жұмыс істейді.

## Неге қызықты

Көптеген инфрақұрылымдық автоматтандырулар уақыт өте келе бір-бірінен алшақтайтын бір реттік скрипттердің үйіндісі ғана. Server Factory керісінше әрекет жасады: модульдік жобалау, орталықтандырылған логтау, ортақ утилиттер, жоғары ағымды басқару сияқты ортақ механизмдерді Core Framework-ке бөліп шығарды да, әрбір құрастыру құралын оның үстінде құра бастады. Осылайша экосистемадағы құралдар біркелкі сезіледі: олар бірдей сәулеттік үлгілерді және сенімді инфрақұрылымды мұраға алады. Бұл серверлерді орналастырудың әдемі емес, бірақ өте маңызды әлемінде қолданылатын фреймворк ойлау тәсілі.

## Қиын мәселелер

Бірнеше әртүрлі құралдардың негізі бола алатын фреймворк құру – жеке құрал құрудан принципті түрде қиынырақ. Серверлерді орналастыратын қолданбалардың ортақ қажеттіліктерін алдын ала болжау керек: біркелкі әрекет ететін логтау, болжамдарды ағытпайтын утилиттер, көптеген қызметтерге сәйкес келетін жоғары ағымды басқару моделі – бірақ осыған тәуелді құралдарды тым шектемеу керек. Абстракцияларды қателессеңіз, оны тұтынатын әрбір құрал қате мұраға алады; дұрыс жасасаңыз, барлығын бірден жеделдетесіз. Экосистема өсе келе осы негізді сақтап, тұтынушыларды тұрақты ұстау – үздіксіз жүргізілетін тәртіп.

## Оны өзгеріске әкелетін нәрсе

Core Framework – бұл күшті құрал. Қиын, ортақ бөліктер бір жерде орналасқандықтан, әрбір жаңа фабрика – пошта, веб-қызметтер, кэш-проксилер, SonarQube және тағы басқалары – бос файлдың орнына сынақтан өткен негізден басталады. Экосистемадағы біркелкілік – бұл төмендеуі мүмкін техникалық қызмет көрсету, аз ғана кенеттен орын алатын жағдайлар және сіз не орналастырсаңыз да біркелкі операциялық тәжірибе. Ашық бастау коды ретінде ол өз серверлік автоматтандыру құралдарын құратын кез келген адамға таза, қайта қолданылатын үлгіні құжаттайды: ортақ қозғалтқышты жеке құралдан бөліп алыңыз.

## Ең қиынырақ бөліктерді қалай шештік

Фреймворк әдейі кесіп өтетін мәселелерді – логтауды, утилиттерді, модульдік құрылымды және жоғары ағымды басқаруды – орталықтандырды, сондықтан жеке фабрикалар оларды қайта жүзеге асырмайды. Осы модульдік жобалау тұтынушыларды біркелкі ұстайтын механизм: олар фреймворк компоненттерін көшірудің орнына оларды құрастырады. Дәлел – экосистеманың өзі. Mail Server Factory және оның туысқан фабрикалары осы негізде тұрады, бұл фреймворк абстракцияларының нақты, өндірістік орналастыру құралдарымен тексерілгенін, ал ойдан шығарылмағанын білдіреді. Фреймворктың құны мүмкіндіктерінде емес, оның үстіндегі құралдарға қаншалықты көп нәрсені өткізіп жіберуге мүмкіндік беретінінде.

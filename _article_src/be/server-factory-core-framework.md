## Кручок
За кожным надзейным інструментам аўтаматызацыі схаваная аснова, якой ніхто не бачыць. Server Factory Core Framework — гэта і ёсць тая аснова: паўторна выкарыстоўваемы рухавік на Kotlin, які сілкуе інструменты накшталт Mail Server Factory, надаючы ўсёй экасістэме Server Factory адзіны, паслядоўны каркас для стварэння серверных прыкладанняў.

## Чаму гэта цікава
Большасць інфраструктурнай аўтаматызацыі — гэта купа разнастайных скрыптоў, якія з часам разыходзяцца ў розныя бакі. Server Factory абраў процілеглы шлях: вынесці агульную механіку — мадульную архітэктуру, цэнтралізаванае лагіраванне, агульныя ўтыліты, кіраванне апстрымам — у Core Framework, а потым будаваць кожны інструмент правізання на яго базе. Вось чаму інструменты экасістэмы ўспрымаюцца як цэласная сістэма: яны атрымліваюць у спадчыну адны і тыя ж архітэктурныя падыходы і аднолькавую надзейную інфраструктуру. Гэта прынцып фрэймворка, прыменены да несамага прывабнага, але крытычна важнага свету разгортвання сервераў.

## Складанасці
Стварэнне фрэймворка, на якім можна будаваць некалькі розных інструментаў, — прынцыпова цяжэйшая задача, чым стварэнне аднаго інструмента. Вам трэба прадбачыць, што патрэбна ўсім прыкладанням для правізання сервераў: лагіраванне, якое паводзіць сябе аднолькава, утыліты, што не навязваюць сваіх здагадак, мадэль кіравання апстрымам, якая падыходзіць для розных сэрвісаў, — і пры гэтым не абмяжоўваць інструменты, што будуць на ім заснаваны. Калі абстракцыі зроблены няправільна — кожны інструмент атрымае гэтую памылку ў спадчыну; калі правільна — вы адразу паскараеце развіццё ўсіх іх. Падтрымка гэтай асновы ў працоўным стане па меры росту экасістэмы, пры захаванні стабільнасці для карыстальнікаў, — гэта пастаянная дысцыпліна.

## Чаму гэта пераварочвае гульню
Core Framework — гэта множнік сілы. Паколькі ўсе складаныя, агульныя кампаненты жывуць у адным месцы, кожны новы завод — для пошты, вэб-сэрвісаў, кэшавых проксі, SonarQube і іншых — пачынае з праверанай базы, а не з пустога файла. Аднолькавасць па ўсёй экасістэме азначае менш абслугоўвання, менш нечаканых сюрпрызаў і адзіны досвед эксплуатацыі, незалежна ад таго, што вы разгортваеце. Як адкрыты зыходны код, ён таксама дэманструе чысты, паўторна выкарыстоўваемы шаблон для тых, хто стварае ўласныя інструменты аўтаматызацыі сервераў: аддзяліце агульны рухавік ад канкрэтнага інструмента.

## Як мы вырашылі найбольш складаныя пытанні
Фрэймворк наўмысна канцэнтруе ў сабе ўсе скрыжаваныя аспекты — лагіраванне, утыліты, мадульную структуру і кіраванне апстрымам, — каб асобныя заводы ніколі не перапісвалі іх нанова. Менавіта гэты мадульны дызайн забяспечвае паслядоўнасць карыстальнікаў: яны кампануюць кампаненты фрэймворка, а не капіяюць іх. Доказ — сама экасістэма. Mail Server Factory і яе сваякі стаяць на гэтай аснове, што азначае: абстракцыі фрэймворка правераны рэальнымі, арыентаванымі на прадукцыйнае выкарыстанне інструментамі правізання, а не выдуманы ў ізаляцыі. Каштоўнасць фрэймворка вымяраецца не колькасцю функцый, а тым, колькі працы ён дазваляе інструментам зверху прапусціць.

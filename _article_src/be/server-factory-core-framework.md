## Хук
За кожным надзейным інструментам аўтаматызацыі стаіць аснова, якой ніхто не бачыць. Server Factory Core Framework — гэта менавіта тая аснова: шматразовы рухавік на Kotlin, які сілкуе інструменты накшталт Mail Server Factory, надаючы ўсёй экасістэме Server Factory адзіную, паслядоўную базу для стварэння серверных прыкладанняў.

## Чаму гэта цікава
Большасць аўтаматызацыі інфраструктуры — гэта купа разнастайных скрыптоў, якія з часам разыходзяцца. Server Factory абраў процілеглы шлях: вынесці агульную машынерыю — мадульную архітэктуру, цэнтралізаванае лагіраванне, агульныя ўтыліты, кіраванне апстрымам — у Core Framework, а потым будаваць кожны інструмент правізання на яго базе. Вось чаму інструменты экасістэмы адчуваюцца як адзінае цэлае: яны атрымліваюць у спадчыну адныя і тыя ж архітэктурныя падыходы і аднолькавую надзейную інфраструктуру. Гэта прынцып фрэймворка, прыменены да няброскай, але крытычна важнай сферы разгортвання сервераў.

## Складанасці
Стварэнне фрэймворка, на якім могуць будавацца некалькі розных інструментаў, прынцыпова складанае за стварэнне аднаго канкрэтнага інструмента. Вам трэба прадбачыць, што патрэбна ўсім прыкладанням для правізання сервераў — лагіраванне з аднолькавым паводзінамі, утыліты без прыхаваных здагадак, мадэль кіравання апстрымам, якая падыходзіць для розных сэрвісаў, — не перагружаючы абмежаваннямі тыя інструменты, якія будуць на ім заснаваны. Памыліцеся з абстракцыямі — і кожны інструмент атрымае гэту памылку ў спадчыну; зрабіце іх правільна — і вы паскорыце развіццё ўсіх адразу. Падтрымка гэтай асновы па меры росту экасістэмы, захоўваючы стабільнасць для карыстальнікаў, — гэта пастаянная дысцыпліна.

## Чаму гэта пераварочвае гульню
Core Framework — гэта множнік. Паколькі ўсе складаныя, агульныя кампаненты жывуць у адным месцы, кожны новы завод — для пошты, вэб-сэрвісаў, кэшавых проксі, SonarQube і іншых — пачынае з праверанай базы, а не з пустога файла. Аднолькавасць па ўсёй экасістэме азначае менш абслугоўвання, менш нечаканых праблем і адзіны досвед эксплуатацыі, незалежна ад таго, што вы разгортваеце. Як адкрыты код, ён таксама дэманструе чысты, шматразовы падыход для тых, хто стварае ўласныя інструменты аўтаматызацыі сервераў: аддзяліце агульны рухавік ад канкрэтнага інструмента.

## Як мы вырашылі найбольш складаныя пытанні
Фрэймворк наўмысна канцэнтруе ў сабе скрыжаваныя аспекты — лагіраванне, утыліты, мадульную структуру і кіраванне апстрымам, — каб асобныя заводы ніколі не перапісвалі іх нанова. Менавіта гэты мадульны дызайн забяспечвае аднолькавасць карыстальнікаў: яны кампануюць кампаненты фрэймворка, а не капіяюць іх. Доказ — сама экасістэма. Mail Server Factory і яе сваякі стаяць на гэтай аснове, што азначае: абстракцыі фрэймворка правераны рэальнымі, вытворчымі інструментамі правізання, а не прыдуманы ў ізаляцыі. Каштоўнасць фрэймворка вымяраецца не колькасцю функцый, а тым, колькі працы ён дазваляе інструментам зверху прапусціць.
